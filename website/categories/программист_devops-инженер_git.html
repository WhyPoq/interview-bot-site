DevOps инженер Git

  1. Как используется git instaweb?

    'git instaweb' используется для автоматического направления веб-браузера и запуска веб-сервера с интерфейсом в вашем локальном репозитории.

  2. Что такое распределенная система контроля версий (VCS)?

    Это системы, которые не зависят от центрального сервера для хранения файла проекта и всех его версий.

  3. Что такое Git fork? В чем разница между fork, branch и clone?

    Форк - это копия репозитория. Обычно вы создаете форк репозитория, чтобы иметь возможность свободно экспериментировать с изменениями, не затрагивая оригинальный проект. В основном форки используются либо для предложения изменений в проекте кого-то другого, либо для использования проекта кого-то другого в качестве отправной точки для своей собственной идеи. Клонирование Git означает указание на существующий репозиторий и создание копии этого репозитория в новом каталоге, в другом месте. Оригинальный репозиторий может находиться на локальной файловой системе или на удаленной машине, доступной по поддерживаемым протоколам. Команда git clone используется для создания копии существующего репозитория Git. Простыми словами, ветки git - это отдельные проекты внутри репозитория git. Различные ветки в репозитории могут иметь совершенно разные файлы и папки, или же все может быть одинаковым, за исключением некоторых строк кода в файле.

  4. Что такое git stash?

    Часто, когда вы работали над частью своего проекта, все находится в беспорядочном состоянии, и вы хотите переключиться на другую ветку, чтобы поработать над чем-то другим. Проблема в том, что вы не хотите делать коммит недоделанной работы, чтобы потом вернуться к этой точке. Решением этой проблемы является Git stash. Stashing берет ваш рабочий каталог, то есть ваши измененные отслеживаемые файлы и поставленные изменения, и сохраняет их в стеке незаконченных изменений, которые вы можете применить в любое время.

  5. Как вы исправите сломанный коммит? Какую команду вы будете использовать?

    Для исправления сломанного коммита в Git мы используем команду “git commit — amend”, которая помогает объединить отслеженные изменения с предыдущими коммитами вместо создания нового коммита.

  6. Как сжать последние N коммитов в один коммит?

    Есть два варианта сжатия последних N коммитов в один коммит, включите оба варианта в вашем ответе Если вы хотите написать новое сообщение коммита с нуля, используйте следующую команду git reset –soft HEAD~N &&git commit Если вы хотите начать редактировать новое сообщение коммита с конкатенацией существующих сообщений коммитов, вам нужно извлечь эти сообщения и передать их в Git commit, для этого я буду использовать git reset –soft HEAD~N &&git commit –edit -m”$(git log –format=%B –reverse .HEAD@{N})”

  7. Сталкивались ли вы с ситуацией, когда вам приходилось разрешать конфликты в Git? Как?

    Конфликт слияния происходит, когда Git не может автоматически разрешить различия в коде между двумя коммитами. Git может автоматически объединить изменения только если коммиты находятся на разных строках или ветках. Вот шаги, которые помогут вам разрешить конфликты в Git: 1. Самый простой способ разрешить конфликтный файл - открыть его и внести все необходимые изменения 2. После редактирования файла мы можем использовать команду git add для добавления нового объединенного содержимого 3. Последний шаг - создать новый коммит с помощью команды git commit 4. Git создаст новый коммит слияния для завершения слияния

  8. Почему желательно создать дополнительный коммит, а не изменить существующий коммит?

    Есть несколько причин для этого - операция изменения уничтожает состояние, которое было сохранено в предыдущем коммите. Если изменяется только сообщение коммита, то это не проблема. Но если изменяются содержимое, то вероятность утери чего-то важного остается выше. Злоупотребление 'git commit --amend' может привести к увеличению небольшого коммита и получению несвязанных изменений.

  9. Что такое SubGit?

    SubGit - это инструмент для миграции с SVN на Git. Он может создать записываемое зеркало Git локального или удаленного репозитория Subversion и использовать как Subversion, так и Git столько, сколько вам угодно.

  10. Объясните разницу между откатом и сбросом.

    Команда Git reset - это мощная команда, которая используется для отмены локальных изменений состояния репозитория Git. Команда Git reset работает с «Тремя деревьями Git», которые представляют собой историю коммитов (HEAD), индексирование и рабочий каталог. Команда revert в Git создает новый коммит, который отменяет изменения предыдущего коммита. Эта команда добавляет новую историю в проект. Она не изменяет существующую историю.

  11. Объясните разницу между перебазированием (rebase) и слиянием (merge) в Git?

    Git rebase - это команда, которая позволяет разработчикам интегрировать изменения из одной ветки в другую. Git merge - это команда, которая позволяет объединить ветки в Git. Git rebase перемещает ветку функции в мастер, в то время как Git merge добавляет новый коммит, сохраняя историю.

  12. Что такое 'git cherry-pick'?

    Эта команда позволяет вам выбирать коммиты из ветки в репозитории и применять их к другой ветке. Эта команда полезна для отмены изменений, когда какой-либо коммит случайно сделан в неправильной ветке. Затем вы можете переключиться на правильную ветку и использовать эту команду для git cherry-pick коммита.

  13. Что такое Head в Git?

    Git поддерживает переменную для ссылки, называемую HEAD, на последний коммит в недавней ветке проверки. Поэтому, если мы делаем новый коммит в репозитории, то указатель или HEAD сместится или изменит свою позицию, чтобы указывать на новый коммит.

  14. Что такое Git?

    Git - это распределенная система контроля версий (DVCS). Он позволяет отслеживать изменения, внесенные в файл, и позволяет вернуться к любому конкретному изменению, которое вы хотите.

  15. В чем разница между 'git diff' и 'git status'?

    'git diff' отображает изменения между коммитами, коммитом и рабочим деревом и т. д., в то время как 'git status' показывает разницу между рабочим каталогом и индексом, что помогает более полно понять git. 'git diff' похож на 'git status', единственное различие в том, что он показывает различия между различными коммитами и также между рабочим каталогом и индексом.
