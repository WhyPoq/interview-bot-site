Backend разработчик SQL Server

  1. Объясните типы репликации в SQL Server.

    Существует три типа репликации: Транзакционная репликация - это процесс распределения данных от издателя к подписчику. Транзакционную репликацию можно использовать, когда данные часто изменяются. Слияние репликации - оно группирует данные в единую централизованную базу данных из различных источников. Слияние репликации используется в случаях, когда центральные и филиальные базы данных должны одновременно обновлять информацию. Снимок репликации - это лучший способ репликации данных, которые изменяются редко, и его легче поддерживать. Пример: Снимок репликации можно использовать для списков, которые обновляются один раз в день и должны распространяться с главного сервера на филиальные серверы.

  2. Что такое альтернативный ключ?

    Когда у таблицы есть более одного кандидатского ключа (т.е. кандидаты на первичные ключи), один становится первичным ключом, а остальные - альтернативными ключами.

  3. Когда следует использовать курсоры, основанные на сервере?

    Когда вам нужно работать с одной записью в любой момент времени, вместо того чтобы брать все данные из таблицы как пакет. Производительность курсоров страдает, когда присутствуют большие объемы данных.

  4. Что такое триггер?

    Когда происходит событие таблицы, такое как INSERT, DELETE или UPDATE, триггеры позволяют выполнить пакет кода SQL. Триггеры управляются СУБД и также могут выполнять хранимые процедуры. Например, когда запись вставляется в таблицу базы данных, можно установить триггер.

  5. Что такое полное резервное копирование?

    Самый распространенный тип резервного копирования в SQL Server - это полное резервное копирование базы данных. Оно также включает часть журналов транзакций для восстановления.

  6. Определите процесс денормализации.

    Процесс денормализации добавляет избыточные данные в базу данных для улучшения производительности. Эта техника переходит от более высоких к более низким нормальным формам базы данных. Это ускоряет доступ к базе данных.

  7. Опишите функцию SIGN.

    Функция SIGN используется для указания числа как положительного, нулевого или отрицательного. Она возвращает следующее: SIGN (число) Возвращает – 1, если число <0, +1, если число>0 и 0, если число=0

  8. Как создать таблицу в SQL?

    Для создания таблицы в SQL используется следующий запрос: Create table name_of_table( column1 datatype, column2 datatype ) Например: create table Student ( Name varchar(20), DOB date, Marks nvarchar(5), Subject varchar(20) )

  10. Как SQL-разработчик, как вы будете обеспечивать хорошую производительность приложений и баз данных на основе SQL Server?

    Необходимо проверить объем данных, тип хранимой информации и данные, к которым нужно обращаться. При обновлении системы следует проанализировать текущие данные и проверить методы доступа к данным, чтобы понять проблему дизайна. Важно сохранять информацию о данных при использовании новой системы.

  11. Что такое целостность ссылочной целостности?

    Каждое значение внешнего ключа должно иметь соответствующее значение первичного ключа. Поддержание этой согласованности между внешними и первичными ключами называется целостностью ссылочной целостности.

  12. В чем разница между rollback и commit?

    Когда выполняется COMMIT, все операторы между BEGIN и COMMIT становятся постоянными для базы данных. В то время как, когда выполняется ROLLBACK, все операторы между ROLLBACK и BEGIN возвращаются к предыдущему состоянию.

  13. Какова важность представлений в базе данных?

    Существуют сценарии, когда нам нужно обратиться к представлению для получения решения, такие как: агрегирование данных для повышения производительности, настройка схемы и данных для набора пользователей, контроль доступа к столбцам и строкам данных.

  14. Что такое пользовательская функция в SQL Server?

    Пользовательская функция позволяет пользователям писать свою логику по мере необходимости. Преимущество заключается в том, что она не ограничивается предопределенными функциями и позволяет писать простой или сложный SQL-код. Тип возвращаемого значения может быть таблицей или скалярным значением.

  15. Что означает столбец идентификатора в SQL?

    В SQL столбец идентификатора генерирует числовые значения автоматически. Эти столбцы не обязательно индексировать, и мы можем определить начальное и инкрементное значение столбца идентификатора.
