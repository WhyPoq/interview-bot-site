Вопросы для собеседования на тему
Frontend разработчик Angular

  1. Что такое Аннотации в Angular?

    Аннотации в Angular используются для создания массива аннотаций. Они представляют собой метаданные, установленные на классе, которые используются для отражения библиотеки метаданных.

  2. Что такое параметризованный pipe?

    Параметризованный pipe в Angular - это pipe, который принимает один или несколько аргументов, также известных как параметры. Пайпы трансформируют данные в шаблонах Angular, и параметризованные пайпы позволяют настраивать трансформацию в соответствии с конкретными требованиями. Передавая параметры в пайп, вы можете изменить его поведение и применить различные трансформации к данным.

  3. Что такое RxJs в Angular?

    RxJs - это библиотека, которая обеспечивает поддержку реактивного программирования для приложений Angular. Она позволяет работать с асинхронными потоками данных и обрабатывать события в течение времени. RxJs основан на Observable, которые являются потоками данных, на которые можно подписаться и обрабатывать с помощью операторов. Он предоставляет мощный и гибкий способ обработки сложных асинхронных операций в Angular.

  4. Какие преимущества и недостатки использования клиентского рендеринга в приложениях Angular?

    Клиентский рендеринг имеет преимущества, такие как улучшенная производительность и лучшая безопасность, но недостатки включают сложности в отладке и потенциальную медленность при обработке большого объема данных с сервера.

  5. Что такое шаблоны в Angular?

    Шаблоны Angular написаны с использованием HTML, который содержит специфические элементы и атрибуты Angular. В сочетании с информацией модели и контроллера, эти шаблоны дополнительно рендерятся для предоставления динамического представления пользователю.

  6. Что такое Bootstrap? Как он встраивается в Angular?

    Bootstrap - это мощный инструментарий. Это набор инструментов HTML, CSS и JavaScript для создания и построения отзывчивых веб-страниц и веб-приложений. Существуют два способа встраивания библиотеки Bootstrap в ваше приложение: Angular Bootstrap через CDN и Angular Bootstrap через NPM.

  7. Что такое ngModule?

    NgModules - это контейнеры, которые резервируют блок кода для домена приложения или рабочего процесса. @NgModule принимает объект метаданных, который обычно описывает способ компиляции шаблона компонента и создания инжектора во время выполнения. Кроме того, он идентифицирует компоненты, директивы и каналы модуля, делая некоторые из них общедоступными через свойство экспорта, чтобы внешние компоненты могли их использовать.

  8. Как включить SASS в проект Angular?

    Чтобы включить SASS в проект Angular, вам нужно установить необходимые зависимости и настроить проект соответственно. Следуйте предоставленным шагам.

  9. Что такое HTTP перехватчики?

    HTTP перехватчики в Angular - это функция, которая позволяет перехватывать HTTP запросы и ответы глобально. Перехватчики предоставляют возможность модифицировать или обрабатывать HTTP запросы и ответы в централизованном месте до их достижения сервера или клиента. Это может быть полезно для ведения журнала запросов, добавления заголовков аутентификации, обработки ошибок или изменения данных запроса/ответа. Перехватчики могут быть зарегистрированы в модуле Angular и выполняются в определенном порядке на основе конвейера запроса/ответа.

  10. Что такое привязка данных? Какой тип привязки данных использует Angular?

    Привязка данных - это явление, которое позволяет любому интернет-пользователю манипулировать элементами веб-страницы с помощью веб-браузера. Она использует динамический HTML и не требует сложного скриптинга или программирования. Мы используем привязку данных на веб-страницах, содержащих интерактивные компоненты, такие как формы, калькуляторы, учебники и игры. Постепенное отображение веб-страницы делает привязку данных удобной, когда на страницах большое количество данных. Angular использует двустороннюю привязку. Любые изменения, внесенные в пользовательский интерфейс, отражаются в соответствующем состоянии модели. Наоборот, любые изменения в состоянии модели отражаются в состоянии пользовательского интерфейса. Это позволяет фреймворку соединить DOM с данными модели через контроллер. Однако, такой подход влияет на производительность, поскольку каждое изменение в DOM должно быть отслежено.

  11. Объясните концепцию внедрения зависимостей.

    Внедрение зависимостей - это техника, используемая для создания слабосвязанного кода. Она позволяет повторно использовать фрагменты кода без необходимости жестко закодированных зависимостей. Это делает код более поддерживаемым и легким для тестирования. Внедрение зависимостей часто используется в фреймворках, таких как AngularJS, ReactJS и VueJS. Также возможно использование внедрения зависимостей в чистом JavaScript. Для использования внедрения зависимостей в JavaScript вам понадобится библиотека внедрения зависимостей. Существует множество различных библиотек, но все они работают примерно одинаково.

  12. Что такое контроллеры?

    Контроллеры AngularJS управляют данными приложений AngularJS. Они являются обычными объектами JavaScript. Директива ng-controller определяет контроллер приложения.

  13. Что такое ngOnInit? Как оно определяется?

    ngOnInit - это хук жизненного цикла и метод обратного вызова, который запускается Angular для указания того, что компонент был создан. Он не принимает параметров и возвращает тип void.

  14. Какой тип DOM реализует Angular?

    DOM (Document Object Model) рассматривает XML- или HTML-документ как древовидную структуру, в которой каждый узел является объектом, представляющим часть документа. Angular использует обычный DOM. Это обновляет всю древовидную структуру HTML-тегов до тех пор, пока не достигнет данных для обновления. Однако, чтобы гарантировать, что скорость и производительность не пострадают, Angular реализует обнаружение изменений (Change Detection).

  15. Что такое router links?

    Router links в Angular используются для навигации в приложении. Они определяются с помощью директивы routerLink и предоставляют способ навигации по различным маршрутам или компонентам. Router links могут использоваться в HTML-шаблонах и обычно размещаются на якорных (<a>) тегах или других элементах, по которым можно кликнуть. Указывая целевой маршрут или компонент, router links позволяют пользователям перемещаться между различными частями приложения Angular.
