Разработчик игр Unity

  1. Что такое обобщения в C# .NET?

    Для уменьшения избыточности кода, повышения безопасности типов и производительности можно использовать обобщения для создания классов кода, которые могут быть повторно использованы. Классы коллекций могут быть созданы с использованием обобщений.

  2. В чем разница между 'throw' и 'throw ex' в .NET?

    'Throw' оператор сохраняет исходный стек ошибок. Но 'throw ex' сохраняет стек вызовов с места исключения.

  3. Объясните четыре этапа компиляции кода на C#.

    Четыре этапа компиляции кода на C# включают в себя - 1. Компиляция исходного кода в управляемом коде. 2. Новый созданный код объединяется с кодом сборки. 3. Загружается среда выполнения общего языка (CLR). 4. Выполнение сборки выполняется через CLR.

  4. Что вы понимаете под свойствами доступа Get и Set?

    В C# Get и Set называются аксессорами, потому что они используют свойства. Такие частные поля доступны через аксессоры.

  5. Что такое события?

    Событие - это уведомление о том, что что-то произошло.

  6. Объясните, почему отложенное освещение оптимизирует сцены с большим количеством источников света и элементов.

    Во время рендеринга каждый пиксель вычисляется, должен ли он быть освещен и получать влияние света, и это повторяется для каждого источника света. После примерно восьми повторных вычислений для различных источников света в сцене накладные расходы становятся значительными. Для больших сцен количество пикселей, отрисованных, обычно больше, чем количество пикселей на экране. Отложенное освещение заставляет сцену отрисовывать все пиксели без освещения (что быстро), и с дополнительной информацией (за счет низких накладных расходов) вычисляет шаг освещения только для пикселей буфера экрана (что меньше, чем все пиксели, обработанные для каждого элемента). Эта техника позволяет использовать гораздо больше источников света в проекте.

  7. Как использовать nullable типы в .Net?

    Когда типы значений могут принимать как обычные значения, так и значение null, их называют nullable типами.

  8. Обсудите важность Asset Bundles в разработке игр на Unity.

    Asset Bundles - это пакеты, содержащие ресурсы (текстуры, модели, звуки и т. д.), которые могут быть загружены динамически во время выполнения. Они позволяют разработчикам уменьшить размер сборки и реализовать потоковую загрузку ресурсов, что является важным для мобильных и онлайн игр, обеспечивая лучший пользовательский опыт и сокращение времени загрузки.

  9. Завершите этот код, чтобы объект GameObject, содержащий этот скрипт, двигался с постоянной скоростью к цели и останавливался, когда достигал 1.0 или менее единиц расстояния.

    class Mover : MonoBehaviour { Vector3 target; float speed; void Update() { float distance = Vector3.Distance(target,transform.position); // будет двигаться только тогда, когда расстояние больше 1.0 единиц if(distance > 1.0f) { Vector3 dir = target - transform.position; dir.Normalize(); transform.position += dir * speed * Time.deltaTime; } } }

  10. В чем разница между Arraylist и Array?

    Массив (Array) содержит только элементы одного типа и его размер фиксирован. ArrayList похож на массив, но у него нет фиксированного размера.

  11. Что такое модификаторы доступа в C#?

    Модификаторы доступа - это термины, которые определяют доступность члена, класса или типа данных программы.

  12. В чем разница между блоками finalize и finalize?

    После завершения блоков try и catch вызывается блок finalize, поскольку он используется для обработки исключений. Независимо от того, было ли исключение перехвачено, этот блок кода будет выполнен. В целом, код в этом блоке более чистый. Прямо перед сборкой мусора вызывается метод finalize. Основные приоритеты метода finalize - это очистка неуправляемого кода, который автоматически запускается, когда экземпляр не вызывается повторно.

  13. В чем разница между операторами “is” и “as” в c#?

    Оператор “is” может использоваться для проверки совместимости объекта с заданным типом, и результат возвращается как логическое значение. Оператор “as” может использоваться для приведения объекта к типу или классу.

  14. Что такое запечатанные классы в C#?

    Когда требуется наложить ограничение на класс, который должен быть унаследован, создаются запечатанные классы. Для предотвращения производных классов используется модификатор sealed. Ошибка времени компиляции возникает, когда запечатанный класс явно указывается как базовый класс.

  15. Что такое мультикастинг делегатов в C#?

    Мультикастинг делегатов позволяет пользователям указывать на более чем один метод в одном вызове.
