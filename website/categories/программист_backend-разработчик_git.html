Вопросы для собеседования на тему

Backend разработчик Git

  1. Как найти список файлов, которые изменились в определенном коммите?

    Используйте `git show --name-only <commit-hash>` для отображения списка файлов, которые изменились в коммите.

  2. Как вы перечислите все удаленные ветки?

    Используйте `git branch -r` для перечисления всех удаленных веток.

  3. Как объединить несколько коммитов в один без слияния?

    Используйте `git rebase --interactive`, чтобы объединить коммиты в один без создания слияния.

  4. Как очистить неотслеживаемые файлы из вашего рабочего каталога?

    Используйте `git clean` для удаления неотслеживаемых файлов из вашего рабочего каталога.

  5. Какова цель `git log --graph`?

    `git log --graph` отображает историю коммитов в графическом представлении.

  6. Что такое `git reflog`?

    `git reflog` показывает журнал перемещений указателей HEAD и веток, позволяя вам вернуться к предыдущим состояниям.

  7. Что такое `git push`?

    `git push` используется для загрузки содержимого локального репозитория в удаленный репозиторий. Он передает коммиты из вашего локального репо на удаленный.

  8. Как настроить Git, чтобы игнорировать изменения в правах доступа к файлам?

    Используйте `git config core.fileMode false`, чтобы игнорировать изменения в правах доступа к файлам.

  9. Какова цель `git gc` (сборщика мусора)?

    `git gc` удаляет ненужные файлы и оптимизирует локальный репозиторий.

  10. Что такое `git reset --soft`?

    `git reset --soft <commit-hash>` отменяет коммиты, но оставляет изменения в области подготовки (staging area).

  11. Что такое быстрое объединение (fast-forward merge) в Git?

    Быстрое объединение происходит, когда конечная точка целевой ветки отстает от конечной точки объединенной ветки, позволяя целевой ветке 'догнать' за счет простого перемещения вперед к конечной точке объединенной ветки.

  12. Как удалить ветку?

    Используйте `git branch -d <branch_name>` для удаления локальной ветки. Если ветка не полностью слита, возможно, вам придется использовать `-D`. Чтобы удалить удаленную ветку, используйте `git push <remote_name> --delete <branch_name>`.

  13. Что такое `git checkout`?

    `git checkout` позволяет перемещаться между различными ветками или возвращать файлы рабочего дерева к предыдущему состоянию. Однако, в последних версиях Git рекомендуется использовать `git switch` для смены веток и `git restore` для возврата файлов, каждая из которых предназначена для выполнения конкретных функций.

  14. Как просмотреть историю коммитов?

    Используйте `git log` для просмотра истории коммитов. Есть много опций для настройки вывода, например, `git log --oneline` для краткого просмотра.

  15. В чем разница между `git merge` и `git rebase`?

    Основное различие заключается в том, как представлена история ветки. `git merge` сохраняет историю ветки функции, создавая новый слияние коммит. `git rebase` переписывает историю ветки функции так, чтобы она выглядела так, будто она была разработана из последней основной ветки, создавая линейную историю.
