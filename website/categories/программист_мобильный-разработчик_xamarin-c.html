Вопросы для собеседования на тему
Мобильный разработчик Xamarin (C#)

  1. Что такое C#?

    C# - это объектно-ориентированный язык программирования, компилируемый с помощью .Net framework для создания промежуточного языка Microsoft.

  2. В чем разница между 'throw' и 'throw ex' в .NET?

    'Throw' оператор сохраняет исходный стек ошибок. Но 'throw ex' сохраняет стек вызовов с места исключения.

  3. Какие различные способы перегрузки метода существуют?

    Различные типы данных могут быть использованы для параметра, чтобы метод был перегружен; также можно использовать различные порядки параметров и различное количество параметров.

  4. Является ли код C# управляемым или неуправляемым кодом?

    C# является управляемым кодом, так как среда выполнения общего языка может компилировать код C# в промежуточный язык.

  5. В чем разница между Arraylist и Array?

    Массив (Array) содержит только элементы одного типа и его размер фиксирован. ArrayList похож на массив, но у него нет фиксированного размера.

  6. Можно ли использовать команду “this” в статическом методе?

    Нет. Это потому что только статические переменные/методы могут быть использованы в статическом методе.

  7. Как можно отсортировать элементы массива в порядке убывания?

    Вы можете использовать метод Sort() и затем метод Reverse().

  8. Что такое индексаторы в C# .NET?

    В C# индексаторы называются умными массивами. Индексаторы позволяют индексировать экземпляры класса так же, как это делают массивы.

  9. Укажите важные среды разработки для C#, предоставленные Microsoft.

    Следующие среды разработки полезны для разработки на C# - MonoDevelop, Visual Studio Code (VS Code), Browxy, Visual Studio Express (VSE), Visual Web Developer (VWD)

  10. Каковы преимущества Xamarin?

    Преимущества Xamarin: - Мы можем создавать нативные приложения для iOS, Android и Windows, используя существующие навыки и код (C#, XAML). - Мы можем полностью использовать нативные API с помощью технологии Xamarin. - Магазин компонентов Xamarin позволяет добавлять высококачественные компоненты в наше приложение непосредственно из среды IDE, включая элементы управления, веб-сервисные API и многое другое. - PCL/shared project упрощает разработку для разработчиков, позволяя им использовать один и тот же кодовую базу в различных проектах.

  11. Что такое облако тестирования Xamarin?

    Облако тестирования Xamarin позволяет нам тестировать мобильное приложение на различных устройствах. Тестовое облако также используется для автоматизированного тестирования на нескольких реальных устройствах одновременно.

  12. Что такое Xamarin Studio?

    Xamarin Studio - это автономная среда разработки для создания кроссплатформенных мобильных приложений, основанная на проекте с открытым исходным кодом Monodevelop.

  13. В чем разница между блоками finalize и finalize?

    После завершения блоков try и catch вызывается блок finalize, поскольку он используется для обработки исключений. Независимо от того, было ли исключение перехвачено, этот блок кода выполняется. В целом, код в этом блоке более чистый. Прямо перед сборкой мусора вызывается метод finalize. Основные приоритеты метода finalize - это очистка неуправляемого кода, который автоматически запускается, когда экземпляр не вызывается повторно.

  14. Что такое объект?

    Объект - это экземпляр класса, который можно использовать для доступа к методам класса. Ключевое слово 'New' можно использовать для создания объекта.

  15. Как работают приложения Xamarin.Android?

    Приложения Xamarin.Android зависят от виртуальной машины Mono от Microsoft. Mono - это открытая реализация .Net Framework от Microsoft, основанная на открытых стандартах C# и CLR. Mono был запущен в 2001 году. Он был создан для того, чтобы .Net-приложения могли работать на платформе Linux, но позже был изменен для поддержки разработки на различных устройствах, включая встроенные системы. В Xamarin Mono работает параллельно с ART Android. На Android большинство системных функций, таких как аудио, графика, OpenGL и телефония, не доступны непосредственно для нативных приложений. Они могут быть добавлены через API Java Android Runtime, доступные в одном из пространств имен Java.* или Android.*. Затем нативные приложения взаимодействуют с открытыми API .NET. Эти API затем, через привязку Android, вызывают API Java Android Runtime.
