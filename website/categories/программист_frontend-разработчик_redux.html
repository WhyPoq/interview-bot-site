  1. Покажите, используя код, как можно использовать константы в Redux.

    Прежде всего, мы можем хранить все константы в одном файле в нашем проекте с именем constants.js или как-то иначе, как показано ниже: export const ADDING_TODO = 'ADDING_TODO'; export const DELETING_TODO = 'DELETING_TODO'; export const EDITING_TODO = 'EDITING_TODO'; export const COMPLETING_TODO = 'COMPLETING_TODO'; export const COMPLETING_ALL = 'COMPLETING_ALL'; export const CLEARING_COMPLETED = 'CLEARING_COMPLETED'; После того, как мы сохранили константы в одном месте, мы можем использовать их двумя способами в нашем проекте: При создании действий (в файле actions.js нашего проекта): import { DELETING_TODO } from './constants'; export function deletingTodo(text) { return { type: DELETING_TODO, text }; } В редукторах (в файле reducer.js нашего проекта): import { EDITING_TODO } from './constants'; export default (state = [], action) => { switch (action.type) { case EDITING_TODO: return [ ...state, { text: action.text, completed: false } ]; default: return state } };

  2. Каковы преимущества использования Redux?

    Некоторые из преимуществ использования Redux следующие: Redux обеспечивает крайне простой перенос состояния между компонентами. Состояния в Redux всегда предсказуемы, и его обслуживание относительно просто. Отладка и тестирование кода в Redux просты благодаря ведению журнала поведения и статуса. Redux обеспечивает отличную производительность. Может показаться, что сохранение глобального состояния приложения приведет к плохой производительности. Однако, обычно это не так, поскольку React Redux внутренне реализует множество оптимизаций производительности, чтобы наш подключенный компонент перерисовывался только тогда, когда это действительно необходимо. Redux также предлагает сохранение состояния путем сохранения состояния приложения в локальном хранилище и восстановления его после обновления.

    Вещи, которые мы никогда не должны делать внутри редьюсера, следующие: изменять аргумент редьюсера, убедиться, что мы не выполняем никаких побочных операций, таких как переходы маршрутизации, вызовы API и т. д., Не вызывать непрозрачные функции, например Date.now(), Math.random(), и т. д.

  4. Выделите основные различия между mapStateToProps() и mapDispatchToProps()?

    Основные различия между mapStateToProps() и mapDispatchToProps() приведены ниже: метод mapStateToProps() используется для отображения хранимых данных в компоненте. Метод mapDispatchToProps() используется для отображения создателей действий с помощью свойств в компоненте. Весь результат метода mapStateToProps() является простым объектом, который затем объединяется в свойство компонента. В методе mapDispatchToProps() каждый создатель действий оборачивается в вызов диспетчера, чтобы их можно было вызывать напрямую и затем объединять в свойство компонента.     Константы в Redux помогают нам легко найти все использования определенной функциональности в нашем проекте, когда мы используем интегрированную среду разработки (IDE). Используя константы, мы можем избежать глупых ошибок, вызванных опечатками или ошибками при наборе, так как они показывают 'ReferenceError' при любой опечатке.

  7. Что вы понимаете под Redux Saga?

    Redux Saga - это библиотека промежуточного программного обеспечения, которая может быть полезна для того, чтобы позволить хранилищу Redux взаимодействовать с ресурсами вне себя асинхронным образом, например, делать HTTP-запросы к внешним службам, получать доступ к хранилищу браузера, выполнять операции ввода/вывода и многое другое. Эти операции также называются побочными эффектами.

  9. Что вы понимаете под Redux Toolkit?

    Redux Toolkit - это официальный, настойчивый, включающий в себя набор инструментов для эффективной разработки Redux. Он также включает в себя наиболее широко используемые дополнения Redux, например, Redux Thunk для асинхронной логики, Reselect для написания функций селекторов и многие другие, чтобы облегчить разработку для разработчиков и сэкономить им время.

  10. Объясните типичный поток данных в приложении, созданном с использованием React и Redux (Жизненный цикл Redux для приложения).

    Типичный поток данных в Redux начинается с обратного вызова из компонента пользовательского интерфейса, который отправляет действие с данными. Затем редьюсеры перехватывают и получают отправленные действия, генерируя новое состояние приложения. После этого действия распространяются через иерархию компонентов из хранилища Redux.

  12. Как можно добавить несколько промежуточных слоев в Redux?

    Для добавления нескольких промежуточных слоев в Redux можно использовать функцию applyMiddleware. В applyMiddleware мы можем передать каждый промежуточный слой как новый аргумент. Наша задача - просто передать каждый промежуточный слой, который нам нужен. В приведенном ниже примере мы добавили промежуточные слои Redux Thunk и logger в качестве аргумента: import { createStore, applyMiddleware } from 'redux' const createStoreWithMiddleware = applyMiddleware(ReduxThunk, logger)(createStore);
