Backend разработчик PostgreSQL

  1. Совместим ли PostgreSQL с облаком?

    Да, PostgreSQL совместим и может быть запущен в облаке. PostgreSQL является очень переносимым. Более того, подобно другим базам данных с открытым исходным кодом, PostgreSQL может легко выполняться в виртуальных контейнерах.

  2. Какой язык запросов является основным для PostgreSQL?

    SQL или язык структурированных запросов является основным языком запросов PostgreSQL.

  3. Перечислите недостатки PostgreSQL.

    Несмотря на множество преимуществ, у PostgreSQL есть много недостатков. Некоторые из них включают: PostgreSQL может иметь более медленную скорость по сравнению с MySQL. Он поддерживает меньше открытых приложений по сравнению с MySQL. Распознаваемость на рынке для PostgreSQL была вызовом из-за отсутствия конкретного владельца. Его производительность может быть ниже, чем у MySQL в определенных ситуациях.

  4. Какова цель утилиты pg_archivecleanup в PostgreSQL?

    Основная цель инструмента pg_archivecleanup в PostgreSQL - избавиться от файлов WAL (Write Ahead Logging) из каталога. Это гарантирует, что будут сохранены только необходимые файлы WAL, необходимые для восстановления в определенный момент времени (PITR).

  5. Какое полное значение ORDBMS?

    Полное значение ORDBMS - это Система управления базами данных объектно-реляционной модели.

  6. Какие типы данных используются в PostgreSQL?

    В PostgreSQL используются следующие типы данных:- Числовой тип данных (Integer, Float) Геометрические примитивы Логический тип данных Тип данных символов (varchar, char, text) Денежный тип данных Массив Тип данных документа (JSON, XML, Key-value, и т. д.) Дата/Время тип данных Тип данных настраиваемый (Композитный, пользовательские типы и т. д.)

  7. Опишите кратко историю PostgreSQL.

    PostgreSQL возник как важная часть проекта POSTGRES, инициированного профессором Майклом Стоунбрейкером в 1986 году в Университете Калифорнии, Беркли. Он совместим с основными операционными системами, такими как macOS, Windows, Linux и UNIX. PostgreSQL поддерживает свойства ACID с 2001 года, с непрерывным развитием ядра платформы на протяжении более трех десятилетий. Он включает значительные дополнения, такие как расширение базы данных PostGIS, и стал стандартной базой данных для macOS, обычно известной как Postgres, благодаря широкому распространению стандарта SQL среди реляционных баз данных.

  8. Опишите метод, с помощью которого можно изменить тип данных столбца в PostgreSQL.

    Тип данных одного или нескольких столбцов в PostgreSQL можно изменить, используя следующие команды вместе с ключевым словом TYPE: ALTER TABLE, ALTER COLUMN. Пример: ALTER TABLE tab_name ALTER COLUMN col_name TYPE new_data_type;

  9. Объясните разделение таблиц в PostgreSQL.

    Разделение таблиц в PostgreSQL - это процесс, при котором большая таблица разбивается на более мелкие части. Эти более мелкие части называются разделами. PostgreSQL поддерживает разделение по списку и диапазону через свою функцию наследования таблиц. Разделение таблиц помогает увеличить производительность запросов в PostgreSQL, так как гораздо проще выбирать данные из этих разделов, чем выбирать из одной основной таблицы. Каждый раздел может хранить данные в соответствии с частотой их использования, что позволяет хранить малоиспользуемые данные на носителях, которые могут быть медленнее или дешевле.

  10. Каковы преимущества логической репликации по сравнению с методами физической репликации?

    Преимущества логической репликации: В логической репликации возможно селективное копирование, что делает ее более гибкой для выбора данных для репликации. Физическая репликация требует идентичных версий PostgreSQL на обоих основных серверах и резервных серверах, в то время как логическая репликация может работать с разными версиями PostgreSQL. Логическая репликация предлагает большую гибкость по сравнению с физической репликацией в обработке изменений в схеме. Логическая репликация совместима с несколькими приложениями и задачами по обслуживанию базы данных.

  11. Какое полное значение MVCC?

    Полное значение MVCC - это многоверсионный контроль одновременности.

  12. Укажите максимальный размер таблицы в PostgreSQL.

    Максимальное количество блоков в таблице определяет предел таблицы. Поскольку количество блоков составляет 2^32, а размер блока по умолчанию равен 8192 байтам, следовательно, максимальный размер таблицы в PostgreSQL составляет 32 ТБ.

  13. Что вы понимаете под командой enable-debug в PostgreSQL?

    Команда enable-debug в PostgreSQL - это команда, которая помогает компилировать все библиотеки и приложения. Она содержит несколько символов отладки, которые облегчают поиск ошибок и других проблем, которые могут возникнуть во время выполнения скрипта. Этот процесс может замедлить или затруднить работу системы при использовании, увеличивая размер двоичного файла.

  14. В чем разница между кластеризованными и не кластеризованными индексами?

    Кластеризованный индекс: Он быстрее, чем не кластеризованный индекс. Индекс считается основными данными в кластеризованном индексе. Кластеризованный индекс имеет возможность естественно хранить данные на диске. Он требует меньше памяти для операций по сравнению с не кластеризованным индексом. В таблице может быть только один кластеризованный индекс. Не кластеризованный индекс: Он относительно медленнее по сравнению с кластеризованным индексом. В случае не кластеризованного индекса, индекс является копией данных. Не кластеризованный индекс не может естественно хранить данные на диске. Не кластеризованный индекс требует больше памяти для выполнения операций. В таблице может быть несколько не кластеризованных индексов.

  15. Как можно решить конфликты репликации в PostgreSQL?

    Когда изменения данных происходят одновременно в разных местах, таких как мульти, что может привести к конфликтам репликации, программное обеспечение автоматически гарантирует, что изменение, которое должно быть разрешено, с помощью «обработчиков конфликтов» в PostgreSQL. Эти обработчики конфликтов автоматически разрешают применимые изменения, чтобы избежать конфликтов.
