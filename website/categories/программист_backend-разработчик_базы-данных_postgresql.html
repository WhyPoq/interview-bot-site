Backend разработчик PostgreSQL

  1. Какое полное значение ORDBMS?

    Полное значение ORDBMS - это Система управления базами данных объектно-реляционной модели.

  2. Какая, на ваш взгляд, является последней версией PostgreSQL на рынке?

    На момент 2022 года последняя версия PostgreSQL на рынке - PostgreSQL 15. Она была запущена 13 октября 2022 года.

  3. Как можно выбрать первые 5 записей в PostgreSQL?

    Ключевое слово LIMIT может быть использовано для выбора первых N записей в PostgreSQL.

  4. Какова цель утилиты pg_archivecleanup в PostgreSQL?

    Основная цель инструмента pg_archivecleanup в PostgreSQL - избавиться от файлов WAL (Write Ahead Logging) из каталога. Это гарантирует, что будут сохранены только необходимые файлы WAL, необходимые для восстановления в определенный момент времени (PITR).

  5. Назовите различные типы операторов, используемых в PostgreSQL.

    Операторы - это специальные символы или слова, которые используются в основном в предложении WHERE в PostgreSQL. Эти операторы могут использоваться для выполнения различных функций и операций. Различные типы операторов, используемых в PostgreSQL, следующие: арифметические операторы, логические операторы, операторы сравнения, побитовые операторы.

  6. Каковы преимущества логической репликации по сравнению с методами физической репликации?

    Преимущества логической репликации: В логической репликации возможно селективное копирование, что делает ее более гибкой для выбора данных для репликации. Физическая репликация требует идентичных версий PostgreSQL на обоих основных серверах и резервных серверах, в то время как логическая репликация может работать с разными версиями PostgreSQL. Логическая репликация предлагает большую гибкость по сравнению с физической репликацией в обработке изменений в схеме. Логическая репликация совместима с несколькими приложениями и задачами по обслуживанию базы данных.

  7. Что такое не кластеризованный индекс?

    В не кластеризованном индексе порядок строк индекса отличается от физического порядка реальных данных. Листовые страницы не кластеризованного индекса содержат указатели на реальные данные, а не сами данные. Его основное преимущество заключается в том, что он обеспечивает более быстрый доступ к данным.

  8. Когда разработчик должен использовать PostgreSQL?

    Разработчики должны выбирать PostgreSQL, когда им требуется надежная и функционально насыщенная система управления базами данных. PostgreSQL подходит для различных приложений, включая веб-разработку, анализ данных и корпоративные решения. Он предоставляет передовые функции, такие как поддержка JSON и геопространственные возможности. Благодаря своей кросс-платформенной совместимости и активному сообществу, разработчики могут полагаться на PostgreSQL для масштабирования и достижения оптимальной производительности при работе с сложными данными.

  9. Перечислите недостатки PostgreSQL.

    Несмотря на множество преимуществ, у PostgreSQL есть много недостатков. Некоторые из них включают: PostgreSQL может иметь более медленную скорость по сравнению с MySQL. Он поддерживает меньше открытых приложений по сравнению с MySQL. Распознаваемость на рынке для PostgreSQL была вызовом из-за отсутствия конкретного владельца. Его производительность может быть ниже, чем у MySQL в определенных ситуациях.

  10. Опишите кратко историю PostgreSQL.

    PostgreSQL возник как важная часть проекта POSTGRES, инициированного профессором Майклом Стоунбрейкером в 1986 году в Университете Калифорнии, Беркли. Он совместим с основными операционными системами, такими как macOS, Windows, Linux и UNIX. PostgreSQL поддерживает свойства ACID с 2001 года, с непрерывным развитием ядра платформы на протяжении более трех десятилетий. Он включает значительные дополнения, такие как расширение базы данных PostGIS, и стал стандартной базой данных для macOS, обычно известной как Postgres, благодаря широкому распространению стандарта SQL среди реляционных баз данных.

  11. Как запустить сервер базы данных в PostgreSQL?

    Для доступа к данным базы данных первым шагом является запуск сервера базы данных. Приложение сервера базы данных, известное как Postgres, играет ключевую роль в управлении базой данных. Оно активно ищет информацию о данных, которые ему необходимо использовать, что является важным для его эффективного функционирования. Инициируя сервер базы данных и предоставляя необходимые сведения о местоположении данных, пользователи могут активно обеспечить доступ и использование данных базы данных через программное обеспечение Postgres. Для этого используется опция -D. Выполните следующие команды для запуска сервера базы данных: usr/local/etc/rc.d/010.pgsql.sh start /usr/local/etc/rc.d/postgresql start Другой способ запустить сервер базы данных в PostgreSQL: Начните с нажатия одновременно клавиш Windows + R, чтобы войти в окно Запуск. Для поиска служб PostgreSQL введите services.msc далее. Используя установленную версию, найдите службу Postgres. Нажмите на кнопку Start, чтобы запустить сервер базы данных.

  12. Каково значение PgAdmin?

    PgAdmin - это бесплатный графический интерфейс с открытым исходным кодом для администрирования баз данных PostgreSQL.

  13. Как можно выполнить восстановление до определенного момента времени в PostgreSQL?

    Для выполнения восстановления до определенного момента времени в PostgreSQL необходимо: Убедитесь, что включено архивирование WAL. Восстановите базовую резервную копию. Примените файлы WAL, либо используя recovery.conf, либо через сервер. Остановите процесс восстановления, когда достигнете желаемой точки восстановления. Разрешите операции на чтение и запись в базе данных.

  14. Объясните термин 'Последовательность' в PostgreSQL.

    Последовательность - это генератор, который производит прогрессивный номер, который может помочь синхронизировать ключи между несколькими строками или таблицами и автоматически создавать один первичный ключ.

  15. Сколько типов репликации существует в PostgreSQL?

    В PostgreSQL существует в основном четыре типа репликации: Физическая репликация (Поточная репликация, Файловая репликация), Логическая репликация (Встроенная логическая репликация, Инструменты сторонних разработчиков), Двунаправленная репликация (BDR), Пользовательские решения по репликации.
