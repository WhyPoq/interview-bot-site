Вопросы для собеседования на тему
Backend разработчик Docker

  1. На каких платформах можно запустить Docker?

    Docker может работать на различных платформах Linux, таких как ArchLinux, CentOS 6+, CRUX 3.0+, Fedora 19/20+, Gentoo, openSUSE 12.3+, RHEL 6.5+, Ubuntu 12.04, 13.04 и др., а также на облачных платформах, таких как Amazon EC2, Amazon ECS, Google Compute Engine, Microsoft Azure, Rackspace.

  2. Объясните виртуализацию.

    Виртуализация - это способ использования программного обеспечения (например, гипервизора) для создания виртуальной версии ресурса, такого как сервер, хранилище данных или приложение. Виртуализация позволяет разделить систему на ряд отдельных секций, каждая из которых действует как отдельная индивидуальная система.

  3. Назовите и объясните различные компоненты Docker.

    Три основных компонента Docker: Docker Client, Docker Host, Registry. Docker Client выполняет операции Docker build pull и run. Docker Host содержит Docker демон, контейнеры и образы. Реестр - это место, где хранятся образы Docker.

  4. Что такое Docker?

    Docker - это платформа контейнеризации с открытым исходным кодом. Она используется для автоматизации развертывания любого приложения с использованием легких, портативных контейнеров.

  5. Назовите основные команды Docker и их назначение.

    Самые важные команды Docker: Build, Commit, Create, Dockerd, Kill. Build создает файл образа Docker, Commit создает новый образ из изменений контейнера, Create создает новый контейнер, Dockerd запускает демон Docker, Kill уничтожает контейнер.

  6. Как создать Docker swarm?

    Используйте следующую команду: docker swarm init –advertise-addr <manager IP>

  7. Опишите жизненный цикл контейнера Docker.

    Обычно жизненный цикл контейнера Docker включает: Создание контейнера, Запуск контейнера, Приостановка контейнера, Возобновление контейнера, Запуск контейнера, Остановка контейнера, Перезапуск контейнера, Уничтожение контейнера.

  8. Может ли контейнер перезапуститься самостоятельно?

    Поскольку флаг по умолчанию -reset установлен в значение false, контейнер не может перезапуститься самостоятельно.

  9. Какая команда для остановки контейнера?

    Используйте следующую команду: $ sudo docker stop <container name>

  10. Можно ли реализовать непрерывное развитие (CD) и непрерывную интеграцию (CI) в Docker?

    Да, вы можете запустить Jenkins на Docker и использовать Docker Compose для запуска интеграционных тестов для непрерывного развития (CD) и непрерывной интеграции (CI).

  11. Что такое контейнер?

    Контейнеры - это развернутые приложения, упакованные со всеми необходимыми зависимостями и файлами конфигурации. Все элементы используют одно и то же ядро ОС. Поскольку контейнер не привязан к какой-либо IT-инфраструктуре, он может работать на другой системе или в облаке.

  12. Какие наиболее заметные особенности Docker?

    Основные особенности Docker включают: Гибкость приложений, Продуктивность разработчиков, Простое моделирование, Операционная эффективность, Размещение и аффинность, Контроль версий.

  13. В чем разница между виртуализацией и контейнеризацией?

    Виртуализация - это абстрактная версия физической машины, в то время как контейнеризация - это абстрактная версия приложения.
