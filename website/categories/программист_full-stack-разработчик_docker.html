Вопросы для собеседования на тему

Full Stack разработчик Docker

  1. Можно ли потерять данные, хранящиеся в контейнере?

    Любые данные, хранящиеся в контейнере, остаются там, пока вы не удалите контейнер.

  2. Что вы знаете о Docker system prune?

    Это команда, используемая для удаления всех остановленных контейнеров, неиспользуемых сетей, кэшей сборки и висячих образов. Prune - одна из самых полезных команд в Docker. Синтаксис: $ docker system prune

  3. Можно ли реализовать непрерывную разработку (CD) и непрерывную интеграцию (CI) в Docker?

    Да, вы можете запустить Jenkins на Docker и использовать Docker Compose для запуска интеграционных тестов для непрерывной разработки (CD) и непрерывной интеграции (CI).

  4. Покажите, как вы бы создали контейнер из образа.

    Для создания контейнера вы загружаете образ из репозитория Docker и запускаете его с помощью следующей команды: $ docker run -it -d <image_name>

  5. Опишите жизненный цикл контейнера Docker.

    Обычно жизненный цикл контейнера Docker включает: Создание контейнера, Запуск контейнера, Приостановка контейнера, Возобновление контейнера, Запуск контейнера, Остановка контейнера, Перезапуск контейнера, Остановка контейнера, Уничтожение контейнера.

  6. Как найти сохраненные тома Docker?

    Используйте команду: /var/lib/docker/volumes

  7. Почему кто-то должен использовать Docker? Что он предлагает?

    Docker дает пользователям много стимулов для принятия, такие как: эффективный и простой опыт начальной настройки, средства для подробного описания жизненного цикла приложения, простая конфигурация и плавное взаимодействие с Docker Compose, полная и подробная документация, возможность запуска на ПК или корпоративной ИТ-системе с равной легкостью.

  8. Как создать Docker swarm?

    Используйте следующую команду: docker swarm init –advertise-addr <manager IP>

  9. Что включает в себя масштабирование контейнера Docker?

    Контейнеры Docker имеют потенциал быть масштабированы до любого необходимого уровня. Благодаря гибкости платформы, вы можете иметь от нескольких сотен до нескольких тысяч, до миллионов контейнеров, при условии, что все они имеют постоянный, неограниченный доступ к необходимой памяти и ОС.

  10. В чем разница между виртуализацией и контейнеризацией?

    Виртуализация - это абстрактная версия физической машины, в то время как контейнеризация - это абстрактная версия приложения.

  11. Какие наиболее заметные особенности Docker?

    Основные особенности Docker включают: Гибкость приложений, Продуктивность разработчиков, Простое моделирование, Операционная эффективность, Размещение и аффинность, Контроль версий.

  12. Объясните виртуализацию.

    Виртуализация - это способ использования программного обеспечения (например, гипервизора) для создания виртуальной версии ресурса, такого как сервер, хранилище данных или приложение. Виртуализация позволяет разделить систему на ряд отдельных секций, каждая из которых действует как отдельная индивидуальная система.

  13. Назовите основные команды Docker и их назначение.

    Самые важные команды Docker: Build, Commit, Create, Dockerd, Kill. Build создает файл образа Docker, Commit создает новый образ из изменений контейнера, Create создает новый контейнер, Dockerd запускает демон Docker, Kill уничтожает контейнер.

  14. Перечислите некоторые более продвинутые команды Docker и их назначение.

    Некоторые продвинутые команды включают: Docker info (отображает системную информацию), Docker pull (загружает образ), Docker stats (предоставляет информацию о контейнере), Docker images (показывает загруженные образы).

  15. Есть ли у Docker какие-либо недостатки?

    Docker не идеален. У него есть свои недостатки, включая: отсутствие опции хранения, неидеальные варианты мониторинга, невозможность автоматического переназначения неактивных узлов, сложная настройка автоматического горизонтального масштабирования.
