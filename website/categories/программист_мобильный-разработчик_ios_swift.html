  1. Что такое кортеж? Как создать кортеж в swift?

    Кортеж - это группа различных значений в одном составном значении. Это упорядоченный список элементов. Есть два способа доступа к данным объекта в кортеже, т.е. по имени или по позиции. В swift кортеж может состоять из нескольких различных типов. Он может содержать два значения, одно целочисленного типа, а другое строкового типа. Это допустимая команда. Например: let ImplementationError = (501, “Not implemented”). Мы можем создать базовый кортеж таким образом: let person = (name: “Ajay” , age: 34)

  2. Какое назначение у двойного вопросительного знака «??» в Swift?

    Двойной вопросительный знак «??» является оператором объединения с nil, это в основном является сокращением для тернарного условного оператора, где мы используем проверку на nil. Двойной вопросительный знак также используется для предоставления значения по умолчанию для переменной. stringVar ?? “default string” Это именно то, что делает обычное действие, если stringVar не nil, то оно возвращается, в противном случае возвращается «default string».

  3. Какие типы объектов являются базовыми типами данных в Swift?

    Swift использует стандартный набор базовых типов данных для различных целей, таких как логические значения, числа и строки.

  4. Что такое модуль Swift?

    Модуль - это единица распространения кода. Фреймворк или приложение создаются и поставляются как единое целое, которое может быть импортировано другим модулем с помощью ключевого слова swift import. Каждая цель сборки в инструменте Xcode рассматривается как отдельный модуль в Swift.

  5. Что такое протокол в Swift?

    Протокол - очень распространенная особенность языка программирования Swift, и протокол - это концепция, аналогичная интерфейсу из Java. Протокол определяет чертеж свойств, методов и других требований, которые подходят для определенной задачи. В своей простейшей форме протокол - это интерфейс, который описывает некоторые методы и свойства. Протокол просто описывает скелет свойств или методов, а не их реализацию. Реализацию свойств и методов можно выполнить, определив перечисления, функции и классы. Протоколы объявляются после имен структуры, перечисления или класса. Возможно объявление одного или нескольких протоколов. Несколько протоколов разделяются запятыми. Мы можем определить протокол таким образом, что он очень похож на структуры, перечисления и классы.

  6. Что такое 'defer'?

    'Defer' - это ключевое слово, которое предоставляет блок кода, который может быть выполнен при выходе выполнения из текущей области видимости.

  7. Объясните Grand Central Dispatch (GDC).

    GCD (Grand Central Dispatch) - это низкоуровневый API для управления параллельными операциями. Этот концепт используется для улучшения производительности приложения. Этот процесс используется для управления несколькими задачами одновременно. Grand Central Dispatch (GCD) является наиболее полезным API для многозадачности с асинхронным и синхронным программированием в iOS.

  8. Что такое PLIST в iOS?

    PLIST означает Property List. PLIST в основном представляет собой словарь значений и ключей, которые могут быть сохранены в нашей файловой системе с расширением файла .plist. Свойственный список используется в качестве портативного и легкого средства для хранения небольшого объема данных. Обычно они записываются в формате XML.

  9. В чем разница между Array и NSArray?

    Разница между Array и NSArray заключается в следующем: Массив может содержать только один тип данных, в то время как NSArray может содержать разные типы данных. Массив является типом значения, в то время как NSArray является неизменяемым типом ссылки.

  12. Как мы можем определить базовый класс в Swift?

    В языке программирования Swift классы не наследуются от базового класса. Классы определяются разработчиком без указания суперкласса, и он автоматически становится базовым классом.

  15. Объясните структуру MVC.

    MVC означает модель представление контроллер. MVC - это мощный шаблон архитектуры программного обеспечения для разработки приложений. MVC строится на основе концепций объектно-ориентированного программирования. Он структурирует поток данных и взаимодействие в нашем приложении. Модель-Представление-Контроллер - это важное фундаментальное понятие, которое необходимо понимать в разработке под iOS. Многие iOS-фреймворки, такие как UIKit, Cocoa Touch, используют шаблон MVC для обмена сообщениями и структурированного потока данных. Модель-Представление-Контроллер используется для передачи данных из одной части нашего приложения в другую. Это шаблон проектирования, используемый для назначения объектов в приложении на любую из трех ролей: Модель, Представление, Контроллер.
