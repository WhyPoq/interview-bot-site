Вопросы для собеседования на тему
DevOps инженер Облачные платформы

  1. Какие контейнеры поддерживает Azure DevOps?

    Azure DevOps предоставляет следующую поддержку контейнеров: Docker, Kubernetes, Azure Container Instances (ACI), Azure Container Registry (ACR)

  2. Назовите основные компоненты AWS, применяемые в DevOps

    Каждый из сервисов AWS разработан для работы в соответствии с культурой DevOps. Но, на мой взгляд, наиболее важными являются те, которые позволяют существовать надежной системе CI/CD, включая: AWS CodePipeline, AWS CodeBuild, AWS CodeDeploy и AWS CodeStar.

  3. Какие типы инструментов доступны через платформу Google Cloud?

    Во-первых, Вычисления. Во-вторых, Сети. В-третьих, Хранение и Базы данных. В-четвертых, Искусственный интеллект (AI) / Машинное обучение (ML). В-пятых, Большие данные. В-шестых, Идентификация и Безопасность. Наконец, Инструменты управления

  4. Что такое AWS CodePipeline?

    AWS CodePipeline - это полностью управляемый сервис CI/CD, который позволяет автоматизировать процессы выпуска для быстрых и надежных обновлений приложений и инфраструктуры.

  5. Как можно обеспечить коммуникацию между участниками команды разработки, работающими в разных местах по всему миру, используя Azure DevOps?

    Для такого приложения изоляция участников различных проектных команд в разные коммуникационные каналы является первым критерием. Кроме того, поддержание истории коммуникации в соответствующих каналах необходимо. Эффективная интеграция с Azure DevOps также важна для приложения и позволяет добавлять внешних поставщиков и подрядчиков в проекты. Команды Microsoft реализовали эти требования через предоставление правильных возможностей. Классификация команд позволяет создавать различные каналы пользователями для организации коммуникаций в соответствии с темой. Каждый канал может включать от нескольких до тысяч пользователей. Microsoft Teams предоставляет функцию гостевого доступа, которая позволяет внешним лицам присоединяться к внутренним каналам для проведения встреч, обмена файлами и обмена сообщениями. Это помогает в управлении проектами B2B и может напрямую интегрироваться с Azure DevOps.

  6. Как реализовать IaC с использованием AWS?

    Существует несколько способов, но лучший вариант при работе с AWS - использовать сервис CloudFormation, который предназначен для написания инфраструктуры в виде кода. Существуют также другие варианты, которые довольно хороши и стоит рассмотреть их, если у нас есть мульти-модельные облака, такие как Terraform или Pulumi. Я предпочитаю использовать Terraform вместо CloudFormation.

  7. Какие языки программирования и фреймворки совместимы с AWS CodeBuild?

    AWS CodeBuild предоставляет предварительно настроенные среды для поддерживаемых версий Java, Ruby, Python, Go, Node.js, Android, .NET Core, PHP и Docker. Вы можете создать и загрузить образ Docker в реестр Amazon EC2 Container Registry или реестр Docker Hub, чтобы настроить свою среду, а затем ссылаться на этот пользовательский образ в своем проекте сборки.

  8. Что такое Azure Repos?

    Azure Repos - это система контроля версий, которая может управлять различными версиями кода и самим кодом на протяжении жизненного цикла разработки. Легко отслеживать любые изменения в коде, внесенные другими командами. Он также ведет подробную запись этих изменений и их историю для лучшей координации в команде. Изменения затем объединяются на более позднем этапе. Azure Repos предлагает централизованную систему контроля версий (Team Foundation Version Control) и распределенную систему контроля версий (Git).

  9. Какова основная разница между Terraform и CloudFormation?

    Terraform - это инструмент от Hashicorp, а CloudFormation - от AWS. При сравнении двух инструментов важно понимать, что, в отличие от Terraform, CloudFormation не является открытым исходным кодом и следует дорожной картой функций, установленной Amazon. Terraform является облачно-независимым. Он предоставляет пользователям значительную гибкость и может использоваться для любого типа ресурса, с которым можно взаимодействовать через API.

  10. Что такое AMI?

    Amazon Machine Image (AMI) - это образ, поддерживаемый и поддерживаемый AWS, который предоставляет информацию, необходимую для запуска экземпляра.

  11. Чем занимается профессиональный инженер DevOps в области облачных технологий?

    Профессиональный инженер DevOps в области облачных технологий отвечает за обеспечение эффективности процессов разработки, балансировку надежности сервиса и скорости доставки. Они знают, как разрабатывать конвейеры доставки программного обеспечения, устанавливать и мониторить сервисы, а также управлять и извлекать уроки из инцидентов с использованием платформы Google Cloud.

  12. Что такое проекты Azure DevOps?

    Проект Azure DevOps - это упрощенный способ эффективного внесения существующего кода и репозиториев Git для создания конвейеров непрерывной интеграции и непрерывной доставки в Azure.

  13. Какие основные функции предлагает Memcached?

    Memcached предлагает широкий спектр функций, таких как: токены CAS, обратные вызовы, GetDelayed, бинарный протокол, Igbinary

  14. Что такое облачные вычисления?

    Облачные вычисления - это предоставление ИТ-услуг по требованию с оплатой по мере использования через Интернет. Вместо покупки, эксплуатации и обслуживания физических центров обработки данных и серверов, вы можете арендовать вычислительную мощность, хранилище и базы данных у облачного провайдера, такого как Google, по мере необходимости. Облачный сервис по вычислениям действительно мировой, без региональных или граничных ограничений.

  15. Расскажите, как вы решили проблему, связанную с созданием конвейера DevOps?

    Важно собрать все требования для создаваемого конвейера. Проведите разговор с командами разработки в случае приложения или, в других случаях, с командой, которая требует функциональности, чтобы определить, как будет разделен конвейер и какие инструменты нужны для его работы. Важно сначала попытаться воспроизвести как можно больше конвейера локально, чтобы затем можно было автоматизировать процесс. Наконец, проведите несколько тестов, чтобы убедиться, что концепция рабочего процесса действительно работает.
